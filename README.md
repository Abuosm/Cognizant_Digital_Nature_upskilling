## Topics Covered

### Design Patterns

Implemented behavioral design patterns:

### 1. Strategy Pattern
- Defines a family of algorithms
- Allows selecting behavior at runtime
- Example: Payment system

### 2. Command Pattern
- Encapsulates requests as objects
- Separates sender and receiver
- Example: Home automation system

### 3. MVC Pattern
- Separates application into:
  - Model
  - View
  - Controller
- Example: Student management system

### 4. Dependency Injection
- Reduces dependency between classes
- Improves code flexibility and testing
- Example: Customer service and repository
